# 网络信息

Surge 面板脚本，用于显示本地、入口、落地三段 IP 信息。当前版本不再依赖外部映射库。

## 文件

- `网络信息.sgmodule`：Surge 模块入口。
- `网络信息.js`：面板脚本主体。

## 优化点

- 直连、本地、代理数据源互相独立，弱网下自动降级。
- 无需下载 `映射库.json`，首次运行不再依赖 GitHub 字典连通性。
- 智能并发查询：首个数据源成功后只短暂等待更优字段，避免慢接口拖住面板。
- 增加 `ipwho.is` HTTPS 数据源，优先补入口城市、本地 ASN、运营商组织字段。
- 保留小型内置别名，只补常见英文城市/运营商中文化，不再依赖大映射库。
- 面板不展示测速/查询耗时，只保留路线和三段节点详情，避免误读。
- 位置优先使用数据源中文字段，国家名由 JS 原生 `Intl.DisplayNames` 智能生成。
- 多源字段融合，中文位置、运营商、ASN 会从不同数据源择优合并。
- 面板顶部显示紧凑路线：本地 -> 入口 -> 落地，并为每段自动加国旗。
- 每段按两到三行展示 IP、位置、运营商和 ASN，尽量减少截断。
- 国家/地区冲突时优先采用更可信的位置源，避免香港节点被泛化成中国大陆。
- 直连机/单跳节点会自动折叠入口，只显示 `本地 -> 落地`，并把入口查询结果合并到落地详情。
- 增加 `ipinfo.io`、`iplocation.net` 兜底和常见 ASN/云厂商别名，城市与运营商识别更稳。
- 入口 IP 变化会触发刷新，同一节点短时间内直接走缓存。
- 本地、入口、落地支持分段缓存继承，单个数据源失败时不清空整块面板。
- 面板采用小浮层优先的紧凑排版，信息更多但不挤成灰字墙。
- 查询、缓存状态、渲染分层处理，后续维护和扩展更清晰。
- 地址、运营商、ASN 做统一清洗，输出更稳定。

## 架构

脚本按 `配置 -> 请求 -> 智能格式化 -> 查询 -> 入口识别 -> 缓存 -> UI 渲染` 分层组织。

缓存中保存结构化数据；刷新时优先使用新查询结果，单段失败时只继承对应旧段，但面板不额外显示缓存状态。

## 使用

在 Surge 中安装 `网络信息.sgmodule`，面板会每 3 秒触发一次脚本；脚本内部有 2.5 秒结果缓存，节点切换时会尽快刷新。

说明：面板不再展示延迟或查询耗时；内部仍会并发查询多个 IP 数据源，只把更可信的节点信息展示出来。

模块安装链接：

```text
https://cdn.jsdelivr.net/gh/justin19933/waibucangku/network-info.sgmodule
```

Surge 一键安装：

```text
surge:///install-module?url=https%3A%2F%2Fcdn.jsdelivr.net%2Fgh%2Fjustin19933%2Fwaibucangku%2Fnetwork-info.sgmodule
```

后续仍使用这个固定模块链接更新即可。模块通过 jsDelivr 分发，脚本地址带版本参数；更新版本号后 Surge 会重新拉取脚本，避免继续使用旧缓存。
